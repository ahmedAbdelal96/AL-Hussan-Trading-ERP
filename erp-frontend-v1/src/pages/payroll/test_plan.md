**Payroll System Testing Plan**

## 🔷 المرحلة 1: اختبار البيانات الأساسية

### 1.1 الموظفين (Employees)

- [ ] إنشاء موظف براتب شهري ثابت (MONTHLY)
- [ ] إنشاء موظف براتب يومي (DAILY)
- [ ] إنشاء موظف براتب ساعي (HOURLY)
- [ ] التأكد من حفظ: الراتب الأساسي، نوع الراتب، تاريخ التعيين
- [ ] تعديل بيانات الراتب والتحقق من التحديث

### 1.2 العلاوات (Allowances)

- [ ] إضافة علاوة ثابتة شهرية `FIXED_MONTHLY`
- [ ] إضافة علاوة نسبية من الراتب `PERCENTAGE`
- [ ] إضافة علاوة لمرة واحدة `ONE_TIME`
- [ ] التحقق من: المبلغ، النوع، التكرار، تاريخ البداية/النهاية
- [ ] حذف علاوة والتأكد من Soft Delete

### 1.3 القروض (Loans)

- [ ] إضافة قرض بـ 6 أقساط
- [ ] إضافة قرض بـ 12 قسط
- [ ] التحقق من: المبلغ الكلي، عدد الأقساط، القسط الشهري
- [ ] التأكد من حساب: `monthlyInstallment = totalAmount ÷ totalInstallments`
- [ ] سداد قسط يدوياً والتحقق من تحديث `paidInstallments`

### 1.4 الخصومات (Deductions)

- [ ] إضافة خصم ثابت شهري `FIXED_MONTHLY`
- [ ] إضافة خصم نسبي `PERCENTAGE`
- [ ] إضافة خصم لمرة واحدة `ONE_TIME`
- [ ] إضافة خصم غياب `ABSENCE`
- [ ] التحقق من حساب الخصومات حسب النوع

## 🔷 المرحلة 2: اختبار حسابات كشوف المرتبات

### 2.1 إنشاء Payroll Sheet

- [ ] إنشاء كشف مرتبات لشهر كامل
- [ ] التحقق من inclusion كل الموظفين النشطين
- [ ] التأكد من الحالة: `DRAFT`

### 2.2 حسابات Payroll Items (لكل موظف)

**السيناريو 1: موظف براتب 10,000 ريال**

```
الراتب الأساسي:     10,000 ريال
+ علاوة ثابتة:       2,000 ريال
+ علاوة نسبية 10%:   1,000 ريال
= إجمالي العلاوات:   3,000 ريال

- قسط قرض:           500 ريال
- خصم غياب:          300 ريال
= إجمالي الخصومات:   800 ريال
─────────────────────────────
= صافي المرتب:       12,200 ريال ✓
```

**الحسابات المطلوب التحقق منها:**

- [ ] `basicSalary` = الراتب من جدول Employees
- [ ] `totalAllowances` = مجموع كل العلاوات النشطة
- [ ] `totalDeductions` = مجموع الخصومات + أقساط القروض
- [ ] `netSalary = basicSalary + totalAllowances - totalDeductions`
- [ ] `status = PENDING`

### 2.3 سيناريوهات متعددة

**سيناريو A: موظف بدون علاوات/خصومات**

- [ ] الراتب: 5,000 | العلاوات: 0 | الخصومات: 0 → **الصافي: 5,000 ✓**

**سيناريو B: علاوة نسبية**

- [ ] الراتب: 8,000 | علاوة 15%: 1,200 → **الصافي: 9,200 ✓**

**سيناريو C: عدة قروض**

- [ ] الراتب: 12,000 | قرض 1: 400 | قرض 2: 300 → **الصافي: 11,300 ✓**

**سيناريو D: موظف جديد (منتصف الشهر)**

- [ ] الراتب الشهري: 6,000 | أيام العمل: 15/30 → **الراتب المستحق: 3,000 ✓**

## 🔷 المرحلة 3: اختبار حالات الـ Payroll Workflow

### 3.1 Status Management

- [ ] إنشاء payroll → حالة `DRAFT`
- [ ] Finalize → حالة `FINALIZED` (لا يمكن التعديل)
- [ ] Approve → حالة `APPROVED`
- [ ] Process Payment → حالة `PAID`
- [ ] التحقق من عدم إمكانية التعديل بعد `FINALIZED`

### 3.2 Loan Installments

- [ ] عند Process Payment → تحديث `paidInstallments++` في جدول Loans
- [ ] التحقق من: `paidInstallments = paidInstallments + 1`
- [ ] عند `paidInstallments === totalInstallments` → حالة القرض `COMPLETED`

### 3.3 One-time Allowances/Deductions

- [ ] بعد استخدام علاوة `ONE_TIME` → تعطيلها تلقائياً (`isActive = false`)
- [ ] بعد استخدام خصم `ONE_TIME` → تعطيله تلقائياً (`isActive = false`)

- [ ] عرض كل كشوف المرتبات للموظف

- [ ] عرض تفاصيل: الراتب، العلاوات، الخصومات، الصافي

- [ ] كل القروض النشطة
- [ ] المبلغ المتبقي = `totalAmount - (monthlyInstallment × paidInstallments)`
- [ ] الأقساط المتبقية = `totalInstallments - paidInstallments`
