## ✅ ما يستطيع المستخدم فعله

### في صفحة الأدوار (Roles):

1. ✅ عرض جميع الأدوار
2. ✅ اختيار دور لإدارة صلاحياته
3. ✅ إضافة/إزالة صلاحيات من الدور
4. ✅ البحث والفلترة

### في صفحة الصلاحيات (Permissions):

1. ✅ عرض جميع الصلاحيات
2. ✅ البحث والفلترة
3. ✅ عرض الـ Resources المتاحة
4. ✅ فهم بنية الصلاحيات

### في صفحة وصول المستخدمين (User Access):

1. ✅ تعيين أدوار للمستخدمين
2. ✅ إلغاء أدوار من المستخدمين
3. ✅ منح صلاحيات مخصصة
4. ✅ إلغاء صلاحيات مخصصة
5. ✅ عرض الصلاحيات الفعلية للمستخدم

## ❌ ما لا يستطيع المستخدم فعله

1. ❌ إنشاء أدوار جديدة
2. ❌ تعديل اسم أو وصف الأدوار
3. ❌ حذف الأدوار
4. ❌ إنشاء صلاحيات جديدة
5. ❌ تعديل الصلاحيات
6. ❌ حذف الصلاحيات

## 🔐 الأدوار المحددة مسبقاً (9 أدوار)

```
1. SUPERADMIN       - صلاحيات كاملة
2. ADMIN            - إدارة النظام
3. HR_MANAGER       - مدير الموارد البشرية
4. HR_STAFF         - موظف الموارد البشرية
5. OPS_MANAGER      - مدير العمليات
6. OPS_STAFF        - موظف العمليات
7. FINANCE_MANAGER  - مدير المالية
8. FINANCE_STAFF    - موظف المالية
9. USER             - مستخدم عادي
```

## 📊 بنية الصلاحيات (36 صلاحية)

```
Pattern: {resource}:{action}

Examples:
- users:read, users:write
- employees:read, employees:write
- sites:read, sites:write
- projects:read, projects:write
- payroll:read, payroll:write, payroll:approve
- etc...
```

## 🚀 كيفية الاستخدام

### إدارة صلاحيات الدور:

1. اذهب إلى **RBAC → Roles & Permissions**
2. انقر على الدور في الجدول
3. حدد/ألغِ تحديد الصلاحيات في اللوحة اليمنى
4. اضغط **Save**

### إدارة وصول المستخدم:

1. اذهب إلى **RBAC → User Access**
2. أدخل UUID المستخدم
3. عيّن أدوار أو امنح صلاحيات مخصصة
4. أضف سبب التغيير (للتدقيق)
5. راجع الصلاحيات الفعلية أسفل الصفحة
